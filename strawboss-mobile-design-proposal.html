Current Design — Analysis
What we found after reviewing all screens, components, and the UI token system.

⚠ Problems Identified

  • Login uses plain text "O" / "*" characters instead of proper eye icons
  • No visual hierarchy between primary and secondary actions — all BigButtons look identical in weight
  • 5-tab driver bar is crowded; no visual priority for the active-trip action
  • Status colors are hardcoded across 3+ files instead of using the token system
  • Background (#F3DED8) clashes with white cards — low perceived contrast in bright sunlight
  • Empty states are bare text with no illustration or clear instruction
  • Tab icons have no active background highlight — active vs inactive hard to distinguish outdoors
  • Font sizes in cards (11–13px) are too small for gloved/dirty hands in the field

💡 Field Context Insights

  • Users work outdoors in direct sunlight — contrast is critical
  • Tractor/loader operators often wear gloves — tap targets must be 48px+ minimum
  • Multiple roles (baler, loader, driver) each have a single primary daily action — the UI must surface it immediately
  • Offline-first architecture means sync state must be prominent and reassuring
  • Romanian-language UI — labels should be tested for length overflow
  • Workers may not be tech-savvy — icons + labels always together, never icons alone

✅ What Works Well

  • Color palette is cohesive — forest green + warm brown reads professional and agricultural
  • BigButton height (56px) is appropriate for field use
  • Soft-delete + offline sync architecture already supports the design needs
  • Card-based layout with border-radius:12 gives a friendly, modern feel
  • MaterialCommunityIcons library covers all needed icons
  • Role-separated layouts (driver/baler/loader) are architecturally solid

Design Principles for This Redesign
☀️
Sunlight Legibility
Minimum 4.5:1 contrast ratio everywhere. Dark green headers provide the high-contrast anchor for all key info.
🧤
Glove-Friendly Targets
All interactive elements 52px minimum height. Cards have generous padding. No tiny icon-only buttons.
🎯
One Primary Action
Each role's home screen shows ONE clear call-to-action above the fold. Secondary actions are visually subordinate.
📶
Sync State Always Visible
Offline status is shown as a persistent banner + a connection card on home. Sync time is always displayed.
🏆
Premium Feel
Gradient headers, frosted glass login card, left-accent trip cards, and fluid motion cues elevate the product.
🔤
Icon + Label Always
No icon-only UI. Every tab, button, and action card pairs an icon with a Romanian text label.

Before / After — Key Differences
What changes and why it matters for field workers.
Before (Current)
✕
Login uses plain text "O" and "*" to toggle password — unintuitive for any user
✕
Flat pink background (#F3DED8) for the full login screen — no brand presence or premium feel
✕
Driver home lists trips immediately with no greeting — user has no spatial orientation
✕
All BigButtons look identical — "Înregistrează producție" and "Starea mea" have equal visual weight
✕
Task badges are 11px text — unreadable in bright sunlight or with gloves
✕
Connection status card is at the bottom, below the fold — offline state goes unnoticed
✕
5-tab driver bar equally weights all tabs — "Cursele Mele" should be dominant
✕
QR scanner is just a raw view with minimal context — no role, no machine ID shown
✕
No visual hierarchy for urgent tasks — urgent and normal look the same
After (Proposed)
✓
Login uses a 4-dot PIN pad visual — familiar pattern for workers who use 4-digit PINs daily
✓
Full-screen gradient green login with frosted glass card — premium, branded, recognisable
✓
Personalised hero header shows name + role badge — users immediately know they're in the right view
✓
Primary action is gradient green, secondary is outline white — clear visual hierarchy
✓
Badges are border-radius:8 with 10px bold text + colored left-border on cards — readable at arm's length
✓
Offline state is a persistent purple banner at the top — always visible, distinctive color
✓
Active trip gets a full hero card with progress bar and big CTA — the driver's one job is unmissable
✓
QR card includes role identity, machine ID, and scanner state — full context at a glance
✓
Urgent tasks have a red circular "!" indicator and "URGENT" badge — impossible to miss
Implementation Notes
How to apply this design to the existing codebase — no architecture changes needed.

🔑 Login Screen

  • Replace the "O"/"*" toggle with MaterialCommunityIcons eye/eye-off — already available in the icon library
  • Add a LinearGradient background (expo-linear-gradient) to the login container — single import
  • Show the 4-dot PIN display when username is not an email — PIN users skip the text password flow
  • The tractor logo can use the existing branding SVG from the /branding folder

🚛 Driver Home

  • Promote the first active/in-transit trip to a hero card above the task list
  • Add a gradient LinearGradient to the SafeAreaView header area (48px tall)
  • Show name from useProfile() hook — already available
  • The trip progress bar maps the status enum to a percentage (planned=10%, loading=30%, loaded=50%, in_transit=65%, arrived=80%, delivering=90%, delivered=100%)

🌾 Baler Home

  • Pull today's bale count from the offline SQLite operations table — already synced
  • The production progress bar uses target from the task assignment if available, else a configurable constant
  • Reduce to 4 tabs (Home, Map, Consumables, Profile) — Stats moves into the Home card
  • Replace the status card with the inline pill row — smaller footprint, same info

📦 Loader Home

  • Wrap the QRScanner in the styled card container — the scanner view itself doesn't change
  • Add machine ID from useAuthStore(s => s.assignedMachineId) already available in the store
  • Offline banner is already the OfflineBanner component — just move it above the hero header visually
  • Urgent tasks use task.priority === "urgent" already in the task object — just style it

🎨 Shared Token Updates

  • Add gradient definitions to ui-tokens/native.ts: primaryGradient, brownGradient, blueGradient
  • Add statusBar colors per role: driverBar (#0A5C36), balerBar (#5D4037), loaderBar (#1565C0)
  • Increase card border-radius token from 12 to 16–20 for a more premium feel
  • Add a shadows.xl for hero cards: 0 8px 32px rgba(0,0,0,0.12)

📱 General UX Rules

  • All tap targets must remain ≥ 52px tall — BigButton is already 56px ✓
  • Replace inline color strings with token references across all screen files
  • Section titles: 11px / 700 / uppercase / letter-spacing 0.5px — consistent across all roles
  • Offline banner: always purple (#7C3AED) to be visually distinct from both the green and brown role colors
